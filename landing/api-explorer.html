API 解读

TDEN Public API Explorer

TDEN 网关对外暴露 3 个标准 API。本页把每个端点的 JSON 用人话翻译一遍,**配中英对照解释每个字段是干嘛的**, 底下保留"查看原始 JSON"链接,方便开发者直接复制使用。

TDEN gateway exposes three public APIs. This page renders each one with plain-language field-level annotations in Chinese + English. Raw JSON link preserved at the bottom of each section for developer copy-paste.

📡 OIDC 发现端点 / OIDC Discovery

RFC 8414 standard endpoint — third-party OAuth clients fetch this once at startup
GET https://gateway.tden.network/.well-known/openid-configuration

第三方应用想做"用 TDEN 登录"时,客户端库(比如 @tden/sdk)第一步就拉这个 JSON, 从里面读出 authorize / token / userinfo 各端点 URL,以及网关支持哪些算法、哪些 scope。 这是 OIDC 协议规定的"自动发现"机制。

First call any OIDC client makes — discovers all the endpoint URLs, supported algorithms, and available scopes so it doesn't need to hard-code them. Standard OAuth 2 / OIDC behavior.

🔑 公钥集合 / JWKS

Ed25519 public keys for verifying ID tokens and access receipts
GET https://gateway.tden.network/oauth/jwks

网关用 Ed25519 私钥签 ID Token 和访问回执;**任何人都能拉这里的公钥独立验签**。 这是"用户主权审计"能成立的底层 ── 浏览器、Go SDK、JS SDK 都用这一份公钥验。

The gateway signs ID tokens and access receipts with its Ed25519 private key. Anyone can fetch these public keys to verify signatures independently — browsers, Go SDK, JS SDK all use this same set.

📦 已批准场景包列表 / Approved Scene Packages

All scene packages reviewers have approved — third parties can request these via OIDC scope
GET https://gateway.tden.network/api/scenepackages

"场景包"是 TDEN 把 OIDC scope 升级为多字段 + 用途声明 + 敏感度自动检测 + 链上审批的结构。 每个包描述一类查询场景(KYC、年龄验证、电商发货…);用户授权时按包整体同意。 被审核员审核通过的就出现在这里,可被任何第三方 RP 引用。

"Scene package" is TDEN's evolution of OIDC scopes — instead of opaque scope strings, each package declares concrete fields, purpose, sensitivity tier, and the user consents per-package. Approved packages here can be referenced by any third-party RP.
